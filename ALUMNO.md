\## ¿Qué es Git?



Git es un sistema de control de versiones que permite a los programadores una mejor coordinación y eficiencia en el trabajo.



\## Por qué necesitamos Git



Por que sin Git cualquier trabajo relacionado a código, tomaría mas tiempo de lo necesario y generaría no solo problemas entre compañeros de equipo, si no también en la relación de empresa y cliente.



\## Los 3 conceptos clave de Git



1\. \*\*Repositorio:\*\* Area de trabajo para un equipo con un objetivo compartido.

2\. \*\*Commit:\*\* Avance en mi área de trabajo asignada con disposición a mejoras y criticas constructivas.

3\. \*\*Rama:\*\* Es como mi oficina en una empresa.



\## Los comandos que aprendí



git init: Es para inicializar y sincronizar git con GitHub.



git pull: para actualizar el código a trabajar y perder la menor cantidad de cambios generados por mi equipo.



git log --oneline: Para visualizar los commits hechos por mi.





\## Mi reflexión final



Lo más importante que aprendí de Git es que no hay nada mejor en un trabajo, que la comunicación y sobre todo la disponibilidad de aceptar mejoras por parte de compañeros que asi como yo, solo buscamos hacer las cosas  de la mejor manera posible.
